Floating Assistant Component

Overview

The Floating Assistant is exactly that, a floating panel controlled by the FloatingUI library. The panel is toggled via interacting with a designated "button" (by click or hover). It has no content of it's own apart from the container so it's mainly a controller of visibility and positioning for the content within it. In theory the panel can be either a floating-from-origin type or a full-height flyout. That will depend on the positioning classes and individual styles used on the panel and any other plugins/helpers attached to it. In all likelihood the from-origin and the full height flyout will just be a configuration difference rather than separate convenience components.

A trimmed-down chat application can be dropped into the floating panel and with added plugins can potentially tap into page-scope contexts and actions/functions. Otherwise it acts as a quick-access chat that remains completely out of the way when not needed.

    The same chat components used to build out the floating panel can be used in a flat/expanded view in case a full-sized chat interface is desired. This layout sandwiches the "chat" panel with a conversations panel on one side and a conversation-context options panel on the other. The current example uses a spa-left layout with the larger context being on the left (all convos) and the specific-context being on the right (the current convo and it's options/config).

    Usage in full-sized, lightweight chat clients

    The layout can also be flipped if right-to-left granularity order is desired. In practice the conversations panel can be placed in another tab panel if additional UI sectioning is required.

    Usage in integrated/split-view chat clients

    A flexible middle-ground between the full-sized client and a hovering/mini client would be an integrated or split-view client that allows for the chat client to share the view with the main page content and size-adjusted as desired.

    The chat client and main content share the space so the chat is always in-view without interfering with the navigation of the main content.

    The floating-style tab navigation for the options panel uses a new 2026-baseline CSS module called the Anchor Positioning module, originally implemented in Chromium circa 2024 before eventually becoming baseline available in January 2026. There are a lot of possibilities with this module as far as layering UI elements and having them auto-calculate their sizes based on known-reference elements/containers.
